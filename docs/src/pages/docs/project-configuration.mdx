# Project Configuration

It is possible to specify a project configuration to specify size checks.

## Usage

You can edit the configuration of the project directly in your project settings. You can also specify this configuration programmatically directly in your project.

Several configuration files are supported:

- A `bundle-analyzer.config.js` file that exports an object (recommended).
- A `.bundle-analyzerrc` file, written in YAML or JSON, with optional extensions: .yaml/.yml/.json/.js.
- A "bundle-analyzer" key in your package.json file.

The configuration file will be resolved from the directory you run the command to upload stats.

## Example

**JSON:**

```json
{
  "files": [
    {
      "test": "*",
      "maxSize": "250 kB"
    }
  ]
}
```

**JS:**

```js
// .bundle-analyzer.config.js
module.exports = {
  files: [
    {
      test: '*',
      maxSize: '250 kB',
    },
  ],
}
```

**YAML:**

```yml
# bundle-analyzerrc.yml
files:
  - test: '*'
    maxSize: 250 kB
```

## Config format

```js
{
  // File rules
  "files": [
    {
      // The file should match this glob expression
      "test": "*.js",
      // The max asset size
      "maxSize": "250 kB",
      // The compression used to operate the size check (none, gzip or brotli)
      "compression": "none"
    }
  ]
}
```

## Check several compressions

To check several compression, just add a rule with a specific compression. The following rule will check all JS file without compression and with "gzip" compression:

```js
{
  "files": [
    {
      "test": "*.js",
      "maxSize": "250 kB",
      "compression": "none"
    },
    {
      "test": "*.js",
      "maxSize": "10 kB",
      "compression": "none"
    }
  ]
}
```
